# 五层能力模型

## 概览

五层能力模型是评估智能体能力与任务需求匹配度的框架，从信息获取到自我进化五个维度全面评估能力覆盖情况。

## 模型定义

### L1：信息获取层（权重 20%）

**定义**：获取和处理任务所需原始数据的能力

**评估维度**：
- 信息获取能力：能否获取所需类型的信息（文本、图像、音频、视频等）
- 数据源充足度：是否有足够的数据源支持任务
- 实时性：能否满足任务的时效性要求（静态/实时/预测）

**评分依据**：
- 1.0：完全满足，有充足且实时的数据源
- 0.8：基本满足，数据源充足但实时性略差
- 0.6：部分满足，数据源有限或实时性不足
- 0.4：勉强满足，需要额外补充数据源
- 0.2：严重不足，缺乏关键数据源
- 0.0：完全不满足

**典型短板类型**：
- `knowledge_gap`：领域知识缺失
- `data_source_gap`：数据源缺失
- `realtime_gap`：实时性不足

---

### L2：认知处理层（权重 25%）

**定义**：理解和分析信息、进行逻辑推理和专业知识应用的能力

**评估维度**：
- 分析深度：能否进行深层次的分析和推理
- 逻辑推理：能否进行复杂的逻辑判断和推导
- 专业知识匹配：是否具备任务所需的专业知识

**评分依据**：
- 1.0：完全满足，具备深层分析和专业推理能力
- 0.8：基本满足，分析深度足够但专业知识略不足
- 0.6：部分满足，分析深度适中，专业知识有限
- 0.4：勉强满足，仅能进行表层分析
- 0.2：严重不足，缺乏核心专业知识
- 0.0：完全不满足

**典型短板类型**：
- `knowledge_gap`：专业知识缺失
- `reasoning_gap`：推理能力不足
- `processing_gap`：处理流程不完善

---

### L3：创作呈现层（权重 25%）

**定义**：生成和呈现输出内容的能力

**评估维度**：
- 输出格式：能否生成所需的格式（文本、图像、视频、代码等）
- 呈现效果：输出质量是否符合专业标准
- 多模态支持：能否处理多种媒体类型

**评分依据**：
- 1.0：完全满足，输出格式丰富且质量优秀
- 0.8：基本满足，输出格式齐全但质量一般
- 0.6：部分满足，支持主要格式但缺乏高级特性
- 0.4：勉强满足，仅支持基础格式
- 0.2：严重不足，缺乏关键格式支持
- 0.0：完全不满足

**典型短板类型**：
- `capability_gap`：特定输出格式能力缺失
- `presentation_gap`：呈现质量不足
- `multimodal_gap`：多模态支持不足

---

### L4：协作编排层（权重 15%）

**定义**：分解任务、组合技能、管理执行状态的能力

**评估维度**：
- 任务分解：能否将复杂任务分解为可执行的子任务
- 技能组合：能否合理组合多个技能协同工作
- 状态管理：能否有效管理任务执行状态和错误处理

**评分依据**：
- 1.0：完全满足，具备完善的任务编排和协同能力
- 0.8：基本满足，任务分解合理，技能组合有效
- 0.6：部分满足，能分解任务但协同能力有限
- 0.4：勉强满足，仅能处理简单任务分解
- 0.2：严重不足，缺乏任务编排能力
- 0.0：完全不满足

**典型短板类型**：
- `capability_gap`：技能编排能力缺失
- `integration_gap`：集成能力不足
- `coordination_gap`：协调能力不足

---

### L5：自我进化层（权重 15%）

**定义**：从反馈中学习、优化策略、持续改进的能力

**评估维度**：
- 反馈机制：能否收集和处理执行反馈
- 学习能力：能否从反馈中提取经验教训
- 调优机制：能否基于反馈优化执行策略

**评分依据**：
- 1.0：完全满足，具备完善的反馈和自优化机制
- 0.8：基本满足，能收集反馈但优化能力有限
- 0.6：部分满足，有基本的反馈机制
- 0.4：勉强满足，反馈机制不完善
- 0.2：严重不足，缺乏反馈机制
- 0.0：完全不满足

**典型短板类型**：
- `learning_gap`：学习能力不足
- `optimization_gap`：优化能力缺失
- `adaptation_gap`：自适应能力不足

## 评分规则

### 整体匹配度计算

```
整体匹配度 = Σ(Li评分 × Li权重)
其中 i = 1, 2, 3, 4, 5
```

示例：
```
整体匹配度 = (L1 × 0.20) + (L2 × 0.25) + (L3 × 0.25) + (L4 × 0.15) + (L5 × 0.15)
           = (0.65 × 0.20) + (0.70 × 0.25) + (0.85 × 0.25) + (0.80 × 0.15) + (0.75 × 0.15)
           = 0.13 + 0.175 + 0.2125 + 0.12 + 0.1125
           = 0.75
```

### 短板识别规则

1. **严重短板**：某层评分 < 0.5 且影响核心任务
2. **中等短板**：某层评分 < 0.6 且影响核心任务
3. **轻微短板**：某层评分 < 0.7 且影响核心任务

### 匹配度等级划分

| 匹配度范围 | 等级 | 建议操作 |
|-----------|------|----------|
| [0.85, 1.0] | 优秀 | 直接执行任务 |
| [0.70, 0.85) | 良好 | 可选补齐，提升效果 |
| [0.60, 0.70) | 一般 | 建议补齐 |
| [0.40, 0.60) | 较差 | 需要补齐或降级执行 |
| [0.0, 0.40) | 差 | 重新定义任务或委托专家 |

## 短板识别

### 短板类型定义

| 短板类型 | 英文标识 | 描述 | 典型层级 |
|---------|---------|------|---------|
| 知识缺口 | `knowledge_gap` | 缺乏特定领域的专业知识 | L1, L2 |
| 能力缺口 | `capability_gap` | 缺乏特定的技能或工具 | L3, L4 |
| 数据源缺口 | `data_source_gap` | 缺乏必要的数据源 | L1 |
| 实时性缺口 | `realtime_gap` | 无法满足实时性要求 | L1 |
| 处理缺口 | `processing_gap` | 处理流程不完善 | L2 |
| 推理缺口 | `reasoning_gap` | 推理能力不足 | L2 |
| 呈现缺口 | `presentation_gap` | 输出呈现质量不足 | L3 |
| 多模态缺口 | `multimodal_gap` | 多模态支持不足 | L3 |
| 集成缺口 | `integration_gap` | 技能集成能力不足 | L4 |
| 协调缺口 | `coordination_gap` | 任务协调能力不足 | L4 |
| 学习缺口 | `learning_gap` | 学习能力不足 | L5 |
| 优化缺口 | `optimization_gap` | 优化能力缺失 | L5 |
| 自适应缺口 | `adaptation_gap` | 自适应能力不足 | L5 |

### 短板严重程度

| 严重程度 | 英文标识 | 定义 | 处理优先级 |
|---------|---------|------|-----------|
| 严重 | `critical` | 影响核心任务，必须补齐 | P0 |
| 中等 | `moderate` | 影响任务质量，建议补齐 | P1 |
| 轻微 | `minor` | 不影响核心任务，可选补齐 | P2 |

## 示例

### 示例 1：法律分析任务

**任务特征**：
- 任务类型：深度分析
- 领域：法律
- 时效性：实时
- 数据类型：政策文档

**能力评估**：
```
L1 信息获取层：0.65
  - 信息获取能力：0.8（能获取政策文档）
  - 数据源充足度：0.6（数据源有限）
  - 实时性：0.5（缺乏实时数据源）
  
L2 认知处理层：0.70
  - 分析深度：0.7（深度适中）
  - 逻辑推理：0.8（推理能力强）
  - 专业知识匹配：0.6（法律知识有限）
  
L3 创作呈现层：0.85
  - 输出格式：0.9（格式丰富）
  - 呈现效果：0.8（质量良好）
  - 多模态支持：0.8（支持多种格式）
  
L4 协作编排层：0.80
  - 任务分解：0.8（分解合理）
  - 技能组合：0.8（组合有效）
  - 状态管理：0.8（管理完善）
  
L5 自我进化层：0.75
  - 反馈机制：0.7（机制完善）
  - 学习能力：0.8（学习能力良好）
  - 调优机制：0.7（调优能力适中）

整体匹配度 = (0.65 × 0.20) + (0.70 × 0.25) + (0.85 × 0.25) + (0.80 × 0.15) + (0.75 × 0.15)
           = 0.75
```

**识别的短板**：
```json
[
  {
    "id": "shortage_1",
    "layer": "L1",
    "type": "data_source_gap",
    "description": "缺乏实时政策数据源",
    "severity": "moderate",
    "impact_core": true
  },
  {
    "id": "shortage_2",
    "layer": "L2",
    "type": "knowledge_gap",
    "description": "缺乏法律专业知识",
    "severity": "critical",
    "impact_core": true
  }
]
```
