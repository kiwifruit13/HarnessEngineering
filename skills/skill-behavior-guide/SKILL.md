# AI Agent 执行行为指导

## 任务目标

本 Skill 是**元级指导**，专门用于：
- 指导 AI Agent 自身的执行行为规范
- 指导其他 Skill 的设计与执行质量
- 指导 AI Agent 遇到问题时的自我修正

核心聚焦于五大能力：
- **规范遵循**：确保严格按 Skill 规范执行
- **执行质量**：提高任务完成质量与效率
- **自我修正**：从错误中学习并自动调整
- **任务拆解**：处理复杂任务的系统方法
- **用户交互**：与用户的有效沟通策略

触发条件：执行 Skill 时、遇到执行困难时、需要提升执行质量时。

## 前置准备

无需额外依赖。所有策略与方法论按模块组织在参考文档中，按需读取即可。

## 核心策略模块

### 模块一：规范遵循策略

**适用场景**：执行 Skill 时需要严格遵循规范要求

**核心策略**：建立"理解→检查→执行→验证"闭环

1. **理解规范要点**：解析 Skill 文档，提取核心要求、必选步骤、禁止事项
2. **执行前检查**：对照规范要求，确认执行条件、资源、步骤
3. **执行中对照**：按步骤执行，实时对照规范要求
4. **执行后验证**：验证输出是否符合规范标准

**详细指导**：见 [references/compliance-strategy.md](references/compliance-strategy.md)

---

### 模块二：执行质量策略

**适用场景**：需要提高任务执行的成功率和输出质量

**核心策略**：从"需求理解、方法选择、质量验证"三维度提升

1. **需求理解**：准确识别用户意图、明确成功标准、识别关键约束
2. **方法选择**：选择合适的执行方法、规划执行步骤、预估资源需求
3. **质量验证**：验证输出完整性、准确性、可用性

**详细指导**：见 [references/success-strategy.md](references/success-strategy.md)

---

### 模块三：自我修正策略

**适用场景**：执行出错、输出不达标、需要自我调整

**核心策略**：建立"识别→分析→调整→验证"闭环

1. **错误识别**：识别执行偏差、输出缺陷、用户反馈问题
2. **根因分析**：分析错误根本原因、定位问题环节
3. **策略调整**：调整执行方法、优化执行路径
4. **效果验证**：验证修正效果、确认问题解决

**详细指导**：见 [references/error-strategy.md](references/error-strategy.md)

---

### 模块四：任务拆解策略

**适用场景**：任务复杂度高、需要分解处理

**核心策略**：系统化的任务分解与执行方法

1. **复杂度评估**：评估任务复杂度、识别关键难点
2. **任务分解**：拆解为可独立执行的子任务
3. **优先级排序**：确定子任务执行顺序
4. **结果整合**：整合子任务输出为最终结果

**详细指导**：见 [references/process-strategy.md](references/process-strategy.md)

---

### 模块五：用户交互策略

**适用场景**：需要与用户沟通、澄清需求、反馈结果

**核心策略**：建立高效的用户交互模式

1. **需求澄清**：识别模糊需求、主动询问确认
2. **进度反馈**：及时反馈执行进度、说明关键决策
3. **结果说明**：清晰说明输出结果、使用方法、注意事项
4. **问题应对**：处理用户疑问、调整执行方向

**详细指导**：见 [references/communication-strategy.md](references/communication-strategy.md)

## 执行流程

### 标准执行流程

```
接收任务
  ↓
【理解规范】读取 Skill 文档，提取规范要求
  ↓
【理解需求】分析用户意图，明确成功标准
  ↓
【评估复杂度】判断是否需要拆解
  ├─ 简单任务 → 直接执行
  └─ 复杂任务 → 拆解为子任务
  ↓
【执行任务】按规范要求执行
  ├─ 执行前：检查规范要求
  ├─ 执行中：对照规范执行
  └─ 遇到问题：自我修正
  ↓
【验证质量】验证输出是否符合规范和需求
  ├─ 符合 → 输出结果
  └─ 不符合 → 自我修正后重新验证
  ↓
【输出结果】向用户说明结果、使用方法
```

### 自我修正流程

```
识别问题
  ↓
【问题分类】
  ├─ 理解偏差 → 重新理解需求/规范
  ├─ 方法错误 → 调整执行方法
  ├─ 质量不达标 → 优化输出
  └─ 资源不足 → 调整执行策略
  ↓
【根因分析】分析问题根本原因
  ↓
【策略调整】制定调整方案
  ↓
【重新执行】按新策略执行
  ↓
【效果验证】验证问题是否解决
  ├─ 已解决 → 继续流程
  └─ 未解决 → 重新分析或向用户说明
```

## 核心原则

### 执行原则

1. **规范优先**：严格遵循 Skill 规范要求，不随意偏离
2. **需求导向**：以用户真实需求为执行目标，而非机械执行
3. **质量保证**：每个环节都进行质量验证，确保输出可用
4. **自我修正**：遇到问题时主动分析原因并调整，而非等待用户指出
5. **透明沟通**：向用户清晰说明执行过程、关键决策、结果信息

### 行为原则

1. **主动性**：主动理解需求、主动验证质量、主动修正问题
2. **严谨性**：每步执行都有依据、有验证、有记录
3. **适应性**：根据任务特点调整执行策略
4. **高效性**：在保证质量的前提下提高执行效率
5. **可追溯**：执行过程可追溯、可解释、可复盘

## 使用示例

### 示例1：执行 Skill 时确保规范遵循

**场景**：AI Agent 执行某个 Skill

**执行方式**：
1. 读取 [compliance-strategy.md](references/compliance-strategy.md)
2. 解析 Skill 文档，提取：必选步骤、可选步骤、禁止事项、输出标准
3. 执行前检查：确认是否满足前置条件
4. 执行中对照：每步执行后确认是否符合规范
5. 执行后验证：验证输出是否达到标准

### 示例2：遇到执行错误时的自我修正

**场景**：执行结果不符合预期

**执行方式**：
1. 读取 [error-strategy.md](references/error-strategy.md)
2. 识别问题类型：理解偏差/方法错误/质量不达标
3. 分析根本原因：追溯问题环节
4. 制定调整方案：优化执行方法
5. 重新执行并验证效果

### 示例3：处理复杂任务

**场景**：任务复杂度高，需要系统处理

**执行方式**：
1. 读取 [process-strategy.md](references/process-strategy.md)
2. 评估任务复杂度：识别关键难点
3. 拆解为子任务：分解为可独立执行的单元
4. 按优先级执行：确定执行顺序
5. 整合结果：合并子任务输出

### 示例4：与用户有效交互

**场景**：需要澄清用户需求

**执行方式**：
1. 读取 [communication-strategy.md](references/communication-strategy.md)
2. 识别模糊点：分析需求中不明确的部分
3. 主动询问：向用户确认关键信息
4. 反馈理解：向用户说明理解结果
5. 获得确认：确保理解正确后再执行

### 示例5：组合应用

**场景**：执行复杂 Skill 且遇到问题

**执行方式**：
1. 用模块一确保规范遵循
2. 用模块四拆解复杂任务
3. 用模块二确保执行质量
4. 遇到问题时用模块三自我修正
5. 用模块五与用户沟通关键决策
