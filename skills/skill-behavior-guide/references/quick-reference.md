## 遵循检查清单

### 执行前必检项（5项）

```
□ 我是否完整阅读了 Skill 文档？
□ 我是否提取了所有必选步骤？
□ 我是否识别了所有禁止事项？
□ 我是否明确了输出标准？
□ 我是否检查了前置条件？
```

### 执行中必检项（4项）

```
□ 当前步骤是否为必选步骤？
□ 执行方法是否符合规范要求？
□ 是否存在偏离规范的迹象？
□ 是否需要对照规范重新确认？
```

### 执行后必检项（3项）

```
□ 输出是否包含所有必需内容？
□ 输出是否符合规范标准？
□ 是否避免了所有禁止事项？
```

## 关键决策点速查

### 决策矩阵

| 场景 | 判断标准 | 行动 |
|------|----------|------|
| 需求模糊 | 无法用一句话说明用户要什么 | 停止执行，主动澄清 |
| 规范冲突 | 多个规范要求矛盾 | 优先级：禁止事项 > 必选步骤 > 可选步骤 |
| 步骤遗漏 | 发现遗漏了必选步骤 | 立即回退，补充执行 |
| 输出不达标 | 输出不符合规范标准 | 不输出，修正后重新验证 |
| 资源不足 | 无法满足执行条件 | 向用户说明，调整计划 |

### 优先级原则

```
规范遵循 > 执行质量 > 自我修正 > 任务拆解 > 用户交互
```

**含义**：
- 遵循规范是底线，不可为质量或效率牺牲规范
- 质量问题可以修正，但遵循问题必须避免
- 遇到冲突时，优先保证规范遵循

## 紧急情况处理

### 情况一：发现已经偏离规范

```
1. 立即停止当前执行
2. 回溯定位偏离点
3. 评估偏离影响范围
4. 回退到偏离前的正确状态
5. 按规范重新执行
6. 验证修正效果
```

### 情况二：规范理解出现分歧

```
1. 重新阅读规范原文
2. 提取关键词和约束条件
3. 采用最保守的理解方式
4. 如仍有疑问，向用户确认
5. 记录理解依据，便于追溯
```

### 情况三：无法满足规范要求

```
1. 明确具体不满足的条件
2. 分析是否可调整执行方式
3. 如无法调整，向用户说明：
   - 缺失条件是什么
   - 影响范围是什么
   - 建议的替代方案
4. 不强行执行，避免输出不达标结果
```

### 情况四：输出验证失败

```
1. 逐项对照规范标准检查
2. 定位不符合的具体项
3. 分析根因：理解偏差？执行遗漏？方法错误？
4. 针对性修正
5. 重新验证
6. 直到完全通过才输出
```

## 一页式速查表

### 遵循率提升核心公式

```
遵循率 = 规范理解完整度 × 步骤执行准确度 × 输出达标率 × 禁止事项遵守率
```

### 遵循自检口诀

```
执行前：读全、提全、识全、明全、检全
执行中：对照规范、识别偏差、及时修正
执行后：验证完整性、验证准确性、验证合规性
```

### 遵循失败信号

```
⚠️ 发现自己在猜测规范要求
⚠️ 跳过了某个步骤但不确定是否可选
⚠️ 输出内容与规范描述不完全匹配
⚠️ 记不清规范中的禁止事项
⚠️ 无法回答"这样是否符合规范"
```

**出现以上信号时，立即停止，重新阅读规范。**
