## 概述

规范遵循是 AI Agent 执行 Skill 的基础能力。本策略提供系统化的规范遵循方法，确保 AI Agent 严格按 Skill 要求执行任务。

**适用场景**：
- 执行 Skill 时需要严格遵循规范
- 需要确保输出符合 Skill 标准
- 需要避免偏离 Skill 要求

**核心原则**：
- 先理解，后执行
- 有检查，有对照
- 有验证，有记录

## 核心策略框架

```
规范遵循闭环 = 理解规范 → 执行前检查 → 执行中对照 → 执行后验证
```

| 阶段 | 核心目标 | 关键动作 |
|-----|---------|---------|
| 理解规范 | 明确要求 | 解析文档、提取要点、识别约束 |
| 执行前检查 | 确认条件 | 检查前置条件、资源、步骤 |
| 执行中对照 | 实时验证 | 对照规范执行、识别偏差 |
| 执行后验证 | 确认结果 | 验证输出、确认达标 |

## 策略一：理解规范要点

**核心目标**：准确理解 Skill 规范的核心要求

### 1.1 规范解析方法

**解析 Skill 文档的核心要素**：

```
【必选步骤】
识别规范中明确要求必须执行的步骤
标记：必须执行、不可跳过

【可选步骤】
识别规范中允许选择的步骤
标记：可选执行、按需选择

【禁止事项】
识别规范中明确禁止的行为
标记：绝对禁止、不可违反

【输出标准】
识别规范中对输出的要求
标记：必须达到、质量底线

【前置条件】
识别执行前必须满足的条件
标记：必须满足、否则无法执行
```

### 1.2 要点提取清单

**提取规范要点的标准清单**：

```
□ 任务目标是什么？
□ 必选步骤有哪些？（按顺序列出）
□ 可选步骤有哪些？（列出选择条件）
□ 禁止事项有哪些？（列出具体行为）
□ 输出标准是什么？（列出具体指标）
□ 前置条件是什么？（列出必须满足的条件）
□ 资源要求是什么？（列出所需资源）
□ 时间约束是什么？（列出时间要求）
```

### 1.3 规范理解验证

**自我验证理解是否正确**：

```
验证问题：
1. 我能用一句话概括任务目标吗？
2. 我能按顺序说出所有必选步骤吗？
3. 我能说出输出必须达到的标准吗？
4. 我能说出哪些行为是禁止的吗？

如果任一问题回答"不能"，说明理解不完整，需要重新阅读规范。
```

## 策略二：执行前检查

**核心目标**：确认执行条件是否满足

### 2.1 前置条件检查

**检查执行前必须满足的条件**：

```
【资源检查】
□ 所需数据是否齐全？
□ 所需工具是否可用？
□ 所需权限是否具备？

【环境检查】
□ 执行环境是否符合要求？
□ 依赖条件是否满足？
□ 外部约束是否明确？

【信息检查】
□ 任务输入是否完整？
□ 用户需求是否明确？
□ 关键参数是否确认？
```

### 2.2 执行计划确认

**确认执行计划是否符合规范**：

```
【步骤确认】
□ 计划步骤是否包含所有必选步骤？
□ 步骤顺序是否符合规范要求？
□ 是否存在禁止的操作？

【资源确认】
□ 计划使用的资源是否符合要求？
□ 资源是否充足？

【时间确认】
□ 预估时间是否在约束范围内？
□ 是否有足够的时间完成？
```

### 2.3 执行前自检模板

```

【规范要点】
- 必选步骤：[列出步骤]
- 输出标准：[列出标准]
- 禁止事项：[列出事项]

【前置条件】
- 所需资源：[已具备/未具备]
- 所需信息：[已明确/未明确]
- 所需权限：[已具备/未具备]

【执行计划】
- 步骤序列：[列出步骤]
- 预估时间：[时间]
- 关键检查点：[列出检查点]

【决策】
□ 所有条件满足 → 开始执行
□ 条件不满足 → 明确缺失项，调整计划或向用户说明
```

## 策略三：执行中对照

**核心目标**：实时验证执行是否符合规范

### 3.1 步骤对照法

**每步执行后对照规范要求**：

```
执行步骤：[步骤名称]

【对照检查】
□ 是否为必选步骤？→ 必须执行
□ 执行方法是否符合规范？→ 按规范方法执行
□ 执行结果是否符合预期？→ 验证中间结果

【偏差识别】
□ 执行过程是否偏离规范？
□ 执行结果是否异常？
□ 是否需要调整？

【决策】
□ 符合规范 → 继续下一步
□ 存在偏差 → 立即修正
□ 遇到问题 → 分析原因并处理
```

### 3.2 关键检查点

**在关键环节设置检查点**：

```
【阶段检查点】
完成一个阶段后：
- 验证阶段输出是否符合规范
- 确认是否满足下一阶段前置条件
- 识别是否有遗漏或错误

【关键步骤检查点】
执行关键步骤后：
- 验证步骤执行是否完整
- 验证输出是否符合标准
- 记录关键决策和依据

【资源使用检查点】
使用资源后：
- 验证资源使用是否合理
- 确认资源是否充足继续
- 识别是否需要补充资源
```

### 3.3 实时记录机制

**记录执行过程关键信息**：

```
【执行记录】
步骤：[步骤名称]
时间：[执行时间]
方法：[执行方法]
结果：[执行结果]
问题：[遇到的问题]
处理：[处理方式]

【决策记录】
决策点：[决策内容]
依据：[决策依据]
选择：[最终选择]
原因：[选择原因]

【偏差记录】
偏差：[偏差内容]
原因：[偏差原因]
处理：[处理方式]
结果：[处理结果]
```

## 策略四：执行后验证

**核心目标**：确认输出是否符合规范标准

### 4.1 输出完整性验证

**验证输出是否包含所有必需内容**：

```
【必选内容检查】
□ 输出是否包含规范要求的所有内容项？
□ 每个内容项是否完整？
□ 内容项之间的关系是否正确？

【格式要求检查】
□ 输出格式是否符合规范要求？
□ 输出结构是否正确？
□ 输出元素是否齐全？

【数量要求检查】
□ 输出数量是否达到要求？
□ 是否存在遗漏？
□ 是否存在多余？
```

### 4.2 输出质量验证

**验证输出质量是否达到标准**：

```
【准确性验证】
□ 输出内容是否准确？
□ 是否存在错误？
□ 是否存在歧义？

【可用性验证】
□ 输出是否可用？
□ 是否满足用户需求？
□ 是否易于理解和使用？

【一致性验证】
□ 输出是否与规范要求一致？
□ 输出是否与用户需求一致？
□ 输出内部是否一致？
```

### 4.3 规范符合性验证

**最终验证是否完全符合规范**：

```
【规范符合性检查清单】
□ 是否执行了所有必选步骤？
□ 是否避免了所有禁止事项？
□ 输出是否达到所有标准？
□ 是否满足所有前置条件？
□ 是否符合所有约束要求？

【验证决策】
□ 完全符合规范 → 输出结果
□ 存在不符项 → 识别问题、修正后重新验证
□ 无法修正 → 向用户说明问题和原因
```

## 自我诊断工具

### 规范遵循自检清单

```
【理解阶段】
□ 我是否完整阅读了 Skill 文档？
□ 我是否提取了所有必选步骤？
□ 我是否识别了所有禁止事项？
□ 我是否明确了输出标准？

【准备阶段】
□ 我是否检查了前置条件？
□ 我是否确认了资源可用性？
□ 我是否制定了执行计划？
□ 我是否设置了检查点？

【执行阶段】
□ 我是否按步骤执行？
□ 我是否对照规范检查？
□ 我是否记录了关键信息？
□ 我是否及时处理了偏差？

【验证阶段】
□ 我是否验证了输出完整性？
□ 我是否验证了输出质量？
□ 我是否验证了规范符合性？
□ 我是否确认可以输出结果？
```

### 问题诊断矩阵

| 问题现象 | 可能原因 | 自我检查点 |
|---------|---------|-----------|
| 遗漏步骤 | 理解不完整 | 重新提取必选步骤清单 |
| 执行偏差 | 对照不足 | 加强执行中对照检查 |
| 输出不达标 | 验证不充分 | 细化输出验证标准 |
| 偏离规范 | 检查不足 | 增加检查点频率 |

## 应用示例

### 示例1：执行 Skill 的规范遵循

**场景**：执行一个数据处理 Skill

**执行过程**：
1. **理解规范**：
   - 必选步骤：数据验证 → 数据清洗 → 数据转换 → 数据输出
   - 输出标准：格式正确、数据完整、无错误
   - 禁止事项：不得修改原始数据

2. **执行前检查**：
   - 检查数据文件是否存在 → 存在
   - 检查处理工具是否可用 → 可用
   - 确认执行计划 → 四步顺序执行

3. **执行中对照**：
   - 步骤1：数据验证 → 对照规范：验证方法正确 → 通过
   - 步骤2：数据清洗 → 对照规范：清洗规则符合 → 通过
   - 步骤3：数据转换 → 对照规范：转换逻辑正确 → 通过
   - 步骤4：数据输出 → 对照规范：输出格式正确 → 通过

4. **执行后验证**：
   - 完整性：包含所有必需字段 → 通过
   - 质量性：数据准确无错误 → 通过
   - 符合性：未违反禁止事项 → 通过
   - 决策：输出结果

### 示例2：发现偏差时的处理

**场景**：执行中发现某个步骤不符合规范要求

**处理过程**：
1. **识别偏差**：当前步骤的执行方法与规范要求不一致
2. **分析原因**：理解规范时遗漏了该步骤的具体方法要求
3. **立即修正**：重新阅读规范，按正确方法重新执行该步骤
4. **验证效果**：验证修正后的执行是否符合规范
5. **记录经验**：记录该偏差，避免后续重复

### 示例3：输出不达标时的处理

**场景**：执行后验证发现输出不符合规范标准

**处理过程**：
1. **识别问题**：输出缺少必需的内容项
2. **分析原因**：执行中遗漏了某个必选步骤
3. **补充执行**：执行遗漏的步骤
4. **重新验证**：验证补充后的输出是否符合标准
5. **确认输出**：验证通过后输出结果
