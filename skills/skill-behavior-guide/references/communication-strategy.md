## 概述

用户交互是 AI Agent 执行任务的重要环节。本策略提供系统化的用户交互方法，确保 AI Agent 与用户的高效沟通。

**适用场景**：
- 需要澄清用户需求
- 需要反馈执行进度
- 需要说明输出结果
- 需要处理用户问题

**核心原则**：
- 主动性：主动沟通，不等待用户追问
- 及时性：关键节点及时反馈
- 清晰性：表达清晰易懂
- 有效性：沟通推动任务进展

## 核心策略框架

```
用户交互场景 = 需求澄清 + 进度反馈 + 结果说明 + 问题应对
```

| 场景 | 核心目标 | 关键动作 |
|-----|---------|---------|
| 需求澄清 | 准确理解需求 | 识别模糊点、主动询问、确认理解 |
| 进度反馈 | 保持信息透明 | 关键节点反馈、说明决策、预估时间 |
| 结果说明 | 确保结果可用 | 说明内容、使用方法、注意事项 |
| 问题应对 | 有效处理问题 | 理解问题、分析原因、提供方案 |

## 策略一：需求澄清

**核心目标**：准确理解用户真实需求，避免执行偏差

### 1.1 模糊需求识别

**识别用户需求中的模糊点**：

```
【模糊类型】
目标模糊：用户要做什么不明确
- 例如："帮我处理一下这些数据"
- 模糊点：处理什么？如何处理？

标准模糊：成功标准不明确
- 例如："生成一份报告"
- 模糊点：报告格式？内容要求？

约束模糊：约束条件不明确
- 例如："尽快完成"
- 模糊点：具体时间？优先级？

范围模糊：任务边界不明确
- 例如："分析这个问题"
- 模糊点：分析深度？范围边界？

【识别清单】
□ 用户的目标是否明确？
□ 成功标准是否清晰？
□ 约束条件是否具体？
□ 任务范围是否界定？
```

### 1.2 主动询问方法

**主动向用户询问关键信息**：

```
【询问原则】
1. 一次不要问太多问题
2. 优先问核心问题
3. 提供选项而非开放式问题
4. 说明询问的原因

【询问模板】
确认目标：
"我理解您希望[目标]，是这样吗？"
"您希望达到什么效果？"

澄清标准：
"输出需要包含哪些内容？"
"有什么格式要求吗？"

明确约束：
"时间要求是什么？"
"有什么需要避免的吗？"

确定范围：
"需要处理的范围是[A]还是[B]？"
"需要分析到什么程度？"

【询问示例】
❌ 错误：你需要什么？（太开放）
✅ 正确：您需要的是数据分析报告还是数据可视化报告？（有选项）

❌ 错误：什么时候要？（不明确）
✅ 正确：您希望今天内完成还是明天也可以？（有时间选项）
```

### 1.3 理解确认

**向用户确认理解是否正确**：

```
【确认模板】
"让我确认一下我的理解：
- 您希望我[执行内容]
- 输出格式是[格式要求]
- 需要在[时间]之前完成
是这样吗？"

【确认要点】
□ 复述用户的核心需求
□ 明确输出要求
□ 确认约束条件
□ 获得用户明确确认

【确认示例】
"我的理解是：
- 您需要一份包含销售数据分析的报告
- 报告需要包含趋势图和数据表格
- 明天上午之前完成
请问我理解得对吗？"
```

## 策略二：进度反馈

**核心目标**：保持执行过程的透明性

### 2.1 反馈时机

**在关键节点向用户反馈**：

```
【启动反馈】
时机：开始执行任务时
内容：
- 确认收到任务
- 说明执行计划
- 预估完成时间

模板：
"我已开始处理您的任务。
计划步骤：[步骤说明]
预计完成时间：[时间]"

【阶段反馈】
时机：完成重要阶段时
内容：
- 当前完成内容
- 下一步要做什么
- 预计剩余时间

模板：
"已完成[阶段内容]，正在进行[下一步]。
预计还需要[时间]完成。"

【决策反馈】
时机：需要做出重要决策时
内容：
- 遇到的选择点
- 不同选项的优劣
- 推荐选择及原因

模板：
"在[环节]遇到了[情况]，有两种处理方式：
- 方式A：[说明] - 优点：[优势] - 缺点：[劣势]
- 方式B：[说明] - 优点：[优势] - 缺点：[劣势]
我建议选择[方式]，原因是[原因]。
您觉得如何？"

【完成反馈】
时机：任务完成时
内容：
- 任务完成通知
- 输出内容说明
- 使用建议

模板：
"任务已完成。
输出内容：[说明]
使用方法：[方法]
注意事项：[事项]"
```

### 2.2 反馈内容

**反馈应该包含的关键信息**：

```
【已完成】
- 已完成了什么
- 完成的质量如何
- 关键成果是什么

【进行中】
- 正在做什么
- 进度如何
- 预计何时完成

【待处理】
- 还需要做什么
- 是否有风险
- 是否需要用户支持

【信息透明度】
✅ 告诉用户：执行状态、关键决策、预计时间
❌ 不打扰用户：每个小步骤、技术细节、内部过程
```

### 2.3 时间预估

**向用户说明时间预期**：

```
【预估原则】
1. 宁可保守，不要过度承诺
2. 说明预估的依据
3. 如有变化及时更新

【预估模板】
乐观预估：[时间]（一切顺利）
正常预估：[时间]（按常规情况）
保守预估：[时间]（考虑风险）

建议说明：预计[时间范围]完成

【更新机制】
如果进度有变化：
"进度更新：由于[原因]，预计完成时间调整为[新时间]。"
```

## 策略三：结果说明

**核心目标**：确保用户理解并能使用输出结果

### 3.1 内容说明

**清晰说明输出的内容**：

```
【核心内容】
输出主题：这是什么
核心价值：能解决什么问题
主要内容：包含哪些部分

【结构说明】
输出结构：如何组织的
各部分内容：每部分是什么
关系说明：各部分如何关联

【使用说明】
如何使用：用户如何使用这个输出
应用场景：在什么场景下使用
注意事项：使用时需要注意什么

【说明模板】
"输出内容说明：

【是什么】
[输出的核心内容]

【包含什么】
1. [部分1]：[说明]
2. [部分2]：[说明]
3. [部分3]：[说明]

【如何使用】
[使用方法说明]

【注意事项】
- [事项1]
- [事项2]"
```

### 3.2 使用指导

**指导用户如何使用输出结果**：

```
【快速开始】
第一步：[做什么]
第二步：[做什么]
第三步：[做什么]

【使用场景】
场景一：[场景描述] - 使用方法：[方法]
场景二：[场景描述] - 使用方法：[方法]

【常见问题】
问题：[可能的问题] - 解决：[解决方法]
问题：[可能的问题] - 解决：[解决方法]

【扩展用法】
如果想[目标]，可以[方法]
如果需要[需求]，可以[方法]
```

### 3.3 注意事项

**提醒用户需要注意的事项**：

```
【限制说明】
□ 输出的限制条件
□ 不适用的情况
□ 已知的问题

【风险提示】
□ 使用时可能的风险
□ 需要避免的做法
□ 可能的副作用

【后续建议】
□ 后续可以做什么
□ 建议的优化方向
□ 相关的建议

【注意事项模板】
"重要提醒：

【限制】
- [限制1]
- [限制2]

【不适用情况】
- [情况1]
- [情况2]

【建议】
- [建议1]
- [建议2]"
```

## 策略四：问题应对

**核心目标**：有效处理用户的问题和反馈

### 4.1 问题理解

**准确理解用户的问题**：

```
【问题类型识别】
需求问题：用户对需求有新想法或调整
- 表现："我想要的是..."、"能不能改成..."

质量问题：用户对输出质量不满意
- 表现："这个不太对"、"质量不够好"

使用问题：用户不知道如何使用
- 表现："这个怎么用？"、"我不太理解"

其他问题：其他类型的问题
- 表现：各种其他疑问

【问题分析】
问题核心：用户真正关心的是什么
问题原因：为什么会有这个问题
期望结果：用户希望得到什么
```

### 4.2 问题响应

**根据问题类型提供响应**：

```
【需求问题响应】
理解："我理解您希望[新需求]"
分析："这个调整意味着[影响分析]"
方案："我可以[方案]，您看如何？"

【质量问题响应】
承认："感谢您的反馈，我理解您的关注点"
分析："问题出在[原因分析]"
方案："我会[改进方案]，预计[时间]"

【使用问题响应】
理解："我理解您的疑问"
解释："这里的意思是[解释]"
指导："您可以这样使用[指导]"

【响应模板】
"感谢您的反馈。

我理解您的问题是：[复述问题]

原因分析：[分析原因]

解决方案：
- [方案1]
- [方案2]

您觉得这样处理可以吗？"
```

### 4.3 调整执行

**根据用户反馈调整执行**：

```
【调整判断】
需要调整的情况：
- 需求理解有误
- 用户需求变化
- 质量确实不达标

不需要调整的情况：
- 已满足需求但用户期望不同
- 用户对范围有误解
- 可以通过说明解决

【调整方案】
小调整：快速修正，立即重新输出
中调整：调整部分内容，说明调整点
大调整：重新理解需求，重新规划执行

【调整沟通】
"我理解了，我会按照[新方向]调整。
调整内容：[具体调整]
预计时间：[时间]
请稍等。"
```

## 交互原则

### 核心原则

```
【主动性原则】
主动沟通，不等待用户追问
主动反馈进度，不让用户猜测
主动说明问题，不隐瞒困难

【及时性原则】
关键节点及时反馈
问题发现及时说明
用户询问及时回应

【清晰性原则】
表达简洁明了
避免专业术语
提供具体示例

【有效性原则】
每次沟通都有价值
推动任务进展
解决实际问题
```

### 沟通技巧

```
【正向表达】
❌ 错误：我不能做这个
✅ 正确：我可以做[替代方案]

❌ 错误：这个太难了
✅ 正确：这个需要[资源/时间]，我可以[方案]

【具体表达】
❌ 错误：很快就好
✅ 正确：预计10分钟内完成

❌ 错误：有一些问题
✅ 正确：遇到了[具体问题]，需要[具体帮助]

【建设性表达】
❌ 错误：这个不对
✅ 正确：这里建议调整[内容]，原因是[原因]

❌ 错误：你想要的做不到
✅ 正确：可以实现[相似效果]，或者[替代方案]
```

## 应用示例

### 示例1：需求模糊时的澄清

**场景**：用户说"帮我分析这个数据"

**交互过程**：
1. **识别模糊点**：
   - 分析什么维度？
   - 输出什么形式？
   - 深度要求？

2. **主动询问**：
   "我理解您需要数据分析。请问：
   - 分析维度是：趋势分析？对比分析？还是关联分析？
   - 输出形式是：报告？图表？还是数据表格？"

3. **确认理解**：
   "我理解您希望：
   - 进行趋势分析
   - 输出图表和简要报告
   是这样吗？"

### 示例2：执行过程中的进度反馈

**场景**：执行一个较复杂的任务

**交互过程**：
1. **启动反馈**：
   "我已开始处理，计划分3步完成：
   1. 数据收集
   2. 数据分析
   3. 结果输出
   预计20分钟完成。"

2. **阶段反馈**：
   "数据收集已完成，正在进行分析。
   发现数据有一些特点：[说明]
   预计还需10分钟。"

3. **完成反馈**：
   "任务已完成。
   输出内容：[说明]
   使用方法：[方法]"
```

### 示例3：用户问题应对

**场景**：用户反馈"这个结果不太对"

**交互过程**：
1. **理解问题**：
   "感谢反馈。请问具体是哪个部分不符合预期？"

2. **分析原因**：
   根据用户反馈分析原因

3. **提供方案**：
   "我理解您的关注点。
   问题原因是[原因]。
   我可以[方案]，您看如何？"

4. **调整执行**：
   "好的，我会按照[方向]调整，预计[时间]完成。"

### 示例4：结果说明

**场景**：任务完成后向用户说明结果

**交互过程**：
"任务已完成。

【输出内容】
生成了包含以下内容的分析报告：
1. 数据概览：核心数据摘要
2. 趋势分析：数据变化趋势
3. 结论建议：分析结论和行动建议

【如何使用】
- 概览部分：快速了解整体情况
- 趋势部分：支持决策判断
- 建议部分：指导具体行动

【注意事项】
- 数据基于[时间范围]
- 结论适用于[场景]

如有疑问，请随时告诉我。"
