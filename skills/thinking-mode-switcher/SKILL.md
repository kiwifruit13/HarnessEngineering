# 思考模式切换指南

## 任务目标

本Skill帮助智能体根据任务特征快速选择合适的思考模式，实现性能与质量的最优平衡。

**核心能力**：快速评估 → 选择模式 → 应用策略

## 四种思考模式

### 模式速查表

| 模式 | 思考深度 | 响应时间 | 典型场景 | 输出特征 |
|------|---------|---------|---------|---------|
| **MODE0 直觉反应** | 无思考 | <10ms | 简单查询、状态检查 | 直接答案，<100字 |
| **MODE1 快速启发** | 1层推理 | 100-500ms | FAQ、标准流程 | 简短建议+1-2句解释 |
| **MODE2 结构化推理** | 2-5层 | 1-10s | 问题求解、逻辑分析 | 分步骤结构化输出 |
| **MODE3 深度反思** | 5+层 | 5-60s+ | 创造性任务、重大决策 | 多维度深度分析 |

---

## 快速评估清单（30秒完成）

### 步骤1：判断紧急度（最高优先级）

```
用户是否使用"立即"、"马上"、"紧急"等词？
├─ 是 → 立即采用 MODE0（直觉反应）
└─ 否 → 继续步骤2
```

### 步骤2：判断复杂度

**勾选符合项（每项+分）：**

```
□ 任务描述超过50字 → +0.2
□ 包含"设计"、"架构"、"分析"、"优化" → +0.3
□ 包含"创造性"、"创新" → +0.4
□ 无标准答案或多种解决方案 → +0.3
□ 涉及多个领域知识 → +0.2

总分计算：
- < 0.3 → 低复杂度
- 0.3-0.7 → 中等复杂度
- > 0.7 → 高复杂度
```

### 步骤3：判断重要性

```
用户是否强调"重要"、"关键"、"核心"？
├─ 是 → 优先级设为 8-10
└─ 否 → 优先级设为 4-7（默认）
```

### 步骤4：模式选择（决策树）

```
紧急度=critical？
├─ 是 → MODE0
└─ 否 ↓
    复杂度>0.7 且 优先级≥8？
    ├─ 是 → MODE3
    └─ 否 ↓
        复杂度<0.3？
        ├─ 是 → MODE1
        └─ 否 → MODE2（默认）
```

## 模式应用要点

### MODE0：直觉反应

**核心原则**：直接回答，零推理

**执行方式**：
- 直接给出答案
- 无需解释原因
- 控制在100字以内

**示例**：
- 用户："今天星期几？" → "星期三。"
- 用户："文件大小？" → "2.3MB。"

---

### MODE1：快速启发

**核心原则**：规则推理，简短说明

**执行方式**：
- 应用常见规则或经验
- 1-2句话简要说明
- 给出标准化建议

**示例**：
- 用户："如何提高写作效率？"
- 回答："使用番茄工作法（25分钟专注+5分钟休息），先列大纲再写作。这样能保持专注并提升结构化思维。"

---

### MODE2：结构化推理（默认模式）

**核心原则**：分步推理，展示过程

**执行方式**：
- 分3-5个步骤
- 每步给出明确推理
- 使用结构化格式（Step 1/2/3...）

**示例**：
- 用户："如何设计推荐系统？"
- 回答：
  ```
  Step 1: 需求分析 - 明确业务目标和用户场景
  Step 2: 数据准备 - 收集用户行为数据和物品特征
  Step 3: 算法选择 - 根据数据规模选择协同过滤或内容推荐
  Step 4: 系统架构 - 设计离线训练和在线服务架构
  Step 5: 评估优化 - 定义准确率、召回率等指标
  ```

---

### MODE3：深度反思

**核心原则**：多维分析，充分论证

**执行方式**：
- 至少3个不同视角
- 分析利弊权衡
- 提供深度论证
- 包含元认知反思

**示例**：
- 用户："AI发展应优先考虑技术进步还是伦理约束？"
- 回答：
  ```
  【视角1：技术进步驱动】
  优势：加速创新、提升效率、解决复杂问题
  风险：可能忽视社会影响、伦理风险
  
  【视角2：伦理约束优先】
  优势：保障安全、维护公平、建立信任
  风险：可能限制创新速度
  
  【视角3：平衡策略】
  建立伦理审查机制
  分阶段推进技术应用
  持续监控社会影响
  
  【元认知反思】
  本问题本身存在二元对立假设，实际上技术与伦理并非非此即彼...
  ```

## 核心原则

### ✅ 最佳实践

1. **默认MODE2**：适合大多数任务，平衡速度与质量
2. **快速判断**：使用清单在30秒内完成评估
3. **动态调整**：执行中发现模式不匹配时及时调整
4. **批量统一**：同类型任务使用相同模式

### ❌ 常见错误

1. **过度思考**：简单任务使用MODE3 → 浪费资源
2. **思考不足**：复杂任务使用MODE0 → 质量下降
3. **忽略紧急度**：critical任务仍用MODE2 → 延误时机
4. **僵化执行**：不根据实际情况灵活调整 → 适用性差

## 典型场景示例

### 示例1：简单查询 → MODE0

```
用户："帮我查一下现在几点了？"

评估：
- 紧急度：medium
- 复杂度：0.1（极低）
- 优先级：5

选择：MODE0

执行：直接回答"现在是下午3点15分。"
```

### 示例2：系统设计 → MODE2

```
用户："设计一个推荐系统架构，需要考虑哪些因素？"

评估：
- 紧急度：medium
- 复杂度：0.7（中高）
- 优先级：7
- 未达MODE3标准（复杂度<0.8）

选择：MODE2

执行：
Step 1: 需求分析（用户场景、业务目标）
Step 2: 数据准备（用户行为数据、物品特征）
Step 3: 算法选择（协同过滤、内容推荐、混合方法）
Step 4: 系统架构（离线训练、在线服务）
Step 5: 评估指标（准确率、召回率、多样性）
```

## 注意事项

1. **灵活应用**：决策流程是指导原则，非僵化规则
2. **持续评估**：执行过程中如发现模式不匹配，及时调整
3. **资源意识**：始终关注Token消耗和响应时间
4. **用户期望**：MODE3任务建议提前告知用户需要较长思考时间
