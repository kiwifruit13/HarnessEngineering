## 概览

天赋属性库是为Prompt框架配置天赋的基础资源库。它包含42个核心天赋的详细配置，每个天赋都经过系统化定义，确保可以在不同场景下精准配置。

**使用说明**：
- 每个天赋都有完整的配置卡，包含能力定义、触发条件、配置参数、组合规则、效果标准、示例场景
- 配置时根据框架需求，从库中选择匹配的天赋进行组合
- 每个天赋都有明确的边界和禁忌，避免误用

## 认知维度

### 快速模式识别

**【基本信息】**
- 属性名称：快速模式识别
- 英文对照：Pattern Recognition
- 属性类别：认知维度 > 感知类天赋

**【能力定义】**
- 核心能力：在极短时间内识别问题的核心模式和本质
- 能力表现：
  - 10秒内锁定问题的核心结构
  - 自动忽略无关细节和干扰信息
  - 能识别问题所属的类型和模式
  - 能提取关键要素和关系
- 能力边界：
  - 适用于结构化、有规律的问题
  - 不适用于完全创新、无先例的场景

**【触发条件】**
- 最佳场景：分析已知类型的问题、快速诊断、模式匹配
- 必备条件：需要一定的问题背景知识库
- 禁忌场景：需要深度探索、创新思维、跨领域联想

**【配置参数】**
- 强度等级：8/10
- 激活速度：快（毫秒级响应）
- 资源消耗：低（直觉化处理）
- 稳定性：高（基于已有模式）

**【组合规则】**
- 最佳搭档：本质洞察、抽象概括、决策果断
- 互补天赋：深度思考、创新思维（提供补充视角）
- 冲突天赋：发散思维（可能分散注意力）
- 排斥天赋：细节敏感（可能过度关注细节）

**【效果标准】**
- 输出特征：简洁、准确、直达本质
- 质量标准：能在一句话内概括问题核心
- 验证指标：
  - 概括长度 ≤ 20字
  - 关键要素识别率 ≥ 90%
  - 模式匹配准确率 ≥ 85%

**【示例场景】**
- **场景1（A.P.E框架-ACTION）**：
  - 输入："我想提高工作效率，但总是拖延"
  - 处理：启动快速模式识别 → 识别为"时间管理+行为改变"模式
  - 输出："解决拖延问题，提升效率"

- **场景2（C.O.A.S.T框架-Context）**：
  - 输入："我是一名大三学生，不知道自己适合什么职业"
  - 处理：启动快速模式识别 → 识别为"职业探索+自我认知"模式
  - 输出："帮助学生进行职业方向探索"

- **场景3（R.I.S.E框架-Recognize）**：
  - 输入："我最近发现自己对数据分析很感兴趣"
  - 处理：启动快速模式识别 → 识别为"兴趣识别+优势发现"模式
  - 输出："识别数据分析相关的天赋线索"

### 本质洞察

**【基本信息】**
- 属性名称：本质洞察
- 英文对照：Essential Insight
- 属性类别：认知维度 > 理解类天赋

**【能力定义】**
- 核心能力：透过现象看本质，理解问题的深层意图和核心价值
- 能力表现：
  - 能快速识别问题背后的真正需求
  - 能理解表面问题之下的深层动机
  - 能判断目标的长期价值和战略意义
  - 能区分现象和本质
- 能力边界：
  - 需要一定的问题背景和上下文
  - 不适用于完全未知的全新领域

**【触发条件】**
- 最佳场景：目标澄清、意图分析、价值判断
- 必备条件：需要足够的信息和上下文
- 禁忌场景：信息极度缺失、需要更多探索

**【配置参数】**
- 强度等级：9/10
- 激活速度：中（需要一定分析时间）
- 资源消耗：中（需要深度思考）
- 稳定性：高（基于逻辑和模式）

**【组合规则】**
- 最佳搭档：快速模式识别、战略思维、价值判断
- 互补天赋：情感感知（理解情感动机）、同理心（理解用户感受）
- 冲突天赋：细节敏感（可能被细节迷惑）
- 排斥天赋：发散思维（可能偏离本质）

**【效果标准】**
- 输出特征：深度、准确、有洞察力
- 质量标准：能揭示问题真正的核心
- 验证指标：
  - 洞察深度评分 ≥ 8/10
  - 核心识别准确率 ≥ 90%
  - 用户认可度 ≥ 85%

**【示例场景】**
- **场景1（A.P.E框架-PURPOSE）**：
  - 输入："我想学习Python编程"
  - 处理：启动本质洞察 → 理解真正意图不是"学习Python"而是"提升职业竞争力"
  - 输出："通过学习Python提升职业技能"

- **场景2（E.R.A框架-Expected）**：
  - 输入："我想提高沟通能力"
  - 处理：启动本质洞察 → 理解真正目的是"建立更好的人际关系"
  - 输出："提升人际沟通能力，建立更好的人际关系"

- **场景3（C.A.R.E框架-Care）**：
  - 输入："我很迷茫，不知道该怎么办"
  - 处理：启动本质洞察 → 理解深层需求是"情感支持和方向指引"
  - 输出："提供情感支持，帮助找到前进方向"

### 抽象概括

**【基本信息】**
- 属性名称：抽象概括
- 英文对照：Abstract Generalization
- 属性类别：认知维度 > 理解类天赋

**【能力定义】**
- 核心能力：将复杂信息提炼成简洁、清晰的抽象表达
- 能力表现：
  - 能用简短的语言概括复杂情况
  - 能提取关键要素，忽略次要信息
  - 能形成抽象的概念和模型
  - 能从具体案例中总结出通用规律
- 能力边界：
  - 需要足够的信息量
  - 不适用于需要保留细节的场景

**【触发条件】**
- 最佳场景：总结汇报、概念定义、模式提炼
- 必备条件：需要完整的信息输入
- 禁忌场景：需要保留细节、提供具体指导

**【配置参数】**
- 强度等级：7/10
- 激活速度：快（快速处理）
- 资源消耗：中（需要信息筛选）
- 稳定性：高（基于逻辑）

**【组合规则】**
- 最佳搭档：快速模式识别、本质洞察、简洁表达
- 互补天赋：细节感知（确保不遗漏关键信息）
- 冲突天赋：发散思维（可能影响概括的聚焦性）
- 排斥天赋：过度细节化（与概括相悖）

**【效果标准】**
- 输出特征：简洁、清晰、易理解
- 质量标准：能用最少的字表达最多的信息
- 验证指标：
  - 概括字数 ≤ 原文字数的30%
  - 关键信息保留率 ≥ 90%
  - 理解准确率 ≥ 95%

**【示例场景】**
- **场景1（A.P.E框架-ACTION）**：
  - 输入："我想通过多种方式提高自己的工作效率，比如时间管理、任务优先级排序、减少干扰等"
  - 处理：启动抽象概括 → 提炼核心
  - 输出："提升工作效率"

- **场景2（T.A.G框架-Goal）**：
  - 输入："我的目标是在未来3个月内掌握数据分析的核心技能，包括数据清洗、数据可视化、统计分析等"
  - 处理：启动抽象概括 → 提炼目标
  - 输出："掌握数据分析核心技能"

- **场景3（R.A.C.E框架-Arguments）**：
  - 输入："天赋是先天的、相对稳定的、可以通过练习转化为优势、每个人都有自己的天赋组合"
  - 处理：启动抽象概括 → 形成论点
  - 输出："天赋是可转化的先天优势"

### 逻辑推理

**【基本信息】**
- 属性名称：逻辑推理
- 英文对照：Logical Reasoning
- 属性类别：认知维度 > 理解类天赋

**【能力定义】**
- 核心能力：基于事实和逻辑进行严密的推理分析
- 能力表现：
  - 能识别因果关系
  - 能进行演绎推理和归纳推理
  - 能发现逻辑漏洞和矛盾
  - 能构建完整的逻辑链条
- 能力边界：
  - 需要明确的前提和事实
  - 不适用于情感化、主观性强的场景

**【触发条件】**
- 最佳场景：问题分析、方案论证、漏洞识别
- 必备条件：需要明确的前提和事实
- 禁忌场景：情感支持、创意激发、不确定性高的场景

**【配置参数】**
- 强度等级：8/10
- 激活速度：中（需要推理过程）
- 资源消耗：高（需要深度思考）
- 稳定性：高（基于逻辑）

**【组合规则】**
- 最佳搭档：本质洞察、风险判断、系统思维
- 互补天赋：类比推理（提供不同视角）
- 冲突天赋：直觉判断（可能与逻辑冲突）
- 排斥天赋：情绪化决策（与逻辑相悖）

**【效果标准】**
- 输出特征：严密、有说服力、无漏洞
- 质量标准：推理过程清晰、结论可信
- 验证指标：
  - 逻辑一致性 ≥ 95%
  - 因果关系准确率 ≥ 90%
  - 结论可信度 ≥ 85%

**【示例场景】**
- **场景1（R.A.C.E框架-Arguments）**：
  - 输入：需要论证"天赋需要刻意练习才能转化为优势"
  - 处理：启动逻辑推理 → 构建因果链条
  - 输出："天赋是原材料 → 需要加工 → 刻意练习是加工方法 → 转化为优势是成品"

- **场景2（B.R.O.K.E框架-Observations）**：
  - 输入：用户反馈"不知道自己的天赋"
  - 处理：启动逻辑推理 → 分析原因
  - 输出："天赋识别需要方法 → 用户缺乏方法 → 导致无法识别 → 需要提供工具"

- **场景3（E.R.A框架-Reasoning）**：
  - 输入：为什么需要了解天赋？
  - 处理：启动逻辑推理 → 构建论证
  - 输出："了解天赋 → 认识优势 → 发挥优势 → 提升效率 → 实现价值"

### 类比推理

**【基本信息】**
- 属性名称：类比推理
- 英文对照：Analogical Reasoning
- 属性类别：认知维度 > 理解类天赋

**【能力定义】**
- 核心能力：通过类比和隐喻，用熟悉的概念解释复杂问题
- 能力表现：
  - 能找到两个不同领域之间的相似性
  - 能用简单的比喻解释复杂概念
  - 能将抽象概念具象化
  - 能通过类比启发新的理解
- 能力边界：
  - 需要有足够的领域知识库
  - 类比可能不够精确，需要补充说明

**【触发条件】**
- 最佳场景：概念解释、复杂问题简化、启发式教学
- 必备条件：需要了解用户的背景知识
- 禁忌场景：需要精确性、专业性强的技术解释

**【配置参数】**
- 强度等级：7/10
- 激活速度：快（快速联想）
- 资源消耗：中（需要知识检索）
- 稳定性：中（类比可能因人而异）

**【组合规则】**
- 最佳搭档：本质洞察、清晰表达、简洁表达
- 互补天赋：逻辑推理（提供精确性补充）
- 冲突天赋：过度精确化（可能削弱类比的效果）
- 排斥天赋：复杂化表达（与类比的目的相悖）

**【效果标准】**
- 输出特征：易懂、生动、有启发性
- 质量标准：用户能快速理解复杂概念
- 验证指标：
  - 理解速度提升 ≥ 50%
  - 记忆保留率 ≥ 70%
  - 类比准确度 ≥ 80%

**【示例场景】**
- **场景1（T.R.A.C.E框架-Template）**：
  - 输入：解释"天赋和优势的区别"
  - 处理：启动类比推理 → 找到熟悉的概念
  - 输出："天赋是原材料（如面粉），优势是成品（如面包）。天赋需要通过练习转化为优势。"

- **场景2（A.P.E框架-PURPOSE）**：
  - 输入：为什么要识别天赋？
  - 处理：启动类比推理 → 找到生活场景
  - 输出："识别天赋就像GPS定位，找到起点才能规划最佳路径。"

- **场景3（C.R.I.S.P.E框架-Creative）**：
  - 输入：如何培养天赋？
  - 处理：启动类比推理 → 找到成长场景
  - 输出："培养天赋就像种树，需要合适的土壤、阳光、水，还需要耐心等待。"

### 敏锐观察

**【基本信息】**
- 属性名称：敏锐观察
- 英文对照：Acute Observation
- 属性类别：认知维度 > 感知类天赋

**【能力定义】**
- 核心能力：快速捕捉细节，发现他人忽略的信息
- 能力表现：
  - 能注意到细微的变化和差异
  - 能发现信息中的隐藏线索
  - 能识别异常和特殊之处
  - 能从多个角度收集信息
- 能力边界：
  - 可能过度关注细节，忽略整体
  - 需要平衡与全局感知

**【触发条件】**
- 最佳场景：线索发现、问题诊断、信息收集
- 必备条件：需要有观察对象
-禁忌场景：需要快速概括、高层级分析

**【配置参数】**
- 强度等级：7/10
- 激活速度：中（需要细致观察）
- 资源消耗：中（需要注意力聚焦）
- 稳定性：中（观察结果可能因人而异）

**【组合规则】**
- 最佳搭档：全局感知、快速模式识别、抽象概括
- 互补天赋：细节感知（增强观察能力）
- 冲突天赋：快速决策（观察需要时间）
- 排斥天赋：忽略细节（与观察相悖）

**【效果标准】**
- 输出特征：细致、全面、有发现
- 质量标准：能发现他人忽略的关键信息
- 验证指标：
  - 线索发现率 ≥ 70%
  - 细节捕捉准确率 ≥ 85%
  - 异常识别率 ≥ 75%

**【示例场景】**
- **场景1（R.I.S.E框架-Identify）**：
  - 输入："我不知道自己的天赋是什么"
  - 处理：启动敏锐观察 → 观察用户的语言和描述
  - 输出："注意到你提到'数据分析'和'逻辑推理'，这可能是你的优势领域"

- **场景2（I.C.I.O框架-Collect）**：
  - 输入：需要收集用户的成长经历
  - 处理：启动敏锐观察 → 捕捉关键信息
  - 输出："你在小时候就喜欢拆解玩具，这说明你有探索和动手的天赋"

- **场景3（B.R.O.K.E框架-Observations）**：
  - 输入：用户反馈"练习了但没有进步"
  - 处理：启动敏锐观察 → 观察反馈细节
  - 输出："注意到你提到'没有明确的计划'，这可能是进步缓慢的原因"

### 全局感知

**【基本信息】**
- 属性名称：全局感知
- 英文对照：Global Perception
- 属性类别：认知维度 > 感知类天赋

**【能力定义】**
- 核心能力：从整体上理解系统，把握全局关系
- 能力表现：
  - 能看到事物的整体结构和关系
  - 能理解各部分之间的相互作用
  - 能把握系统的运作机制
  - 能预测系统变化的影响
- 能力边界：
  - 可能忽略重要细节
  - 需要平衡与细节感知

**【触发条件】**
- 最佳场景：系统设计、战略规划、全局分析
- 必备条件：需要有完整的系统信息
- 禁忌场景：需要深度细节分析

**【配置参数】**
- 强度等级：8/10
- 激活速度：中（需要整体把握）
- 资源消耗：中（需要整合信息）
- 稳定性：高（基于系统思维）

**【组合规则】**
- 最佳搭档：敏锐观察、系统思维、战略思维
- 互补天赋：细节感知（补充细节信息）
- 冲突天赋：过度细节化（可能影响全局视角）
- 排斥天赋：局部思维（与全局感知相悖）

**【效果标准】**
- 输出特征：全面、系统、有前瞻性
- 质量标准：能把握整体和关系
- 验证指标：
  - 全局完整度 ≥ 85%
  - 关系识别率 ≥ 80%
  - 预测准确度 ≥ 70%

**【示例场景】**
- **场景1（R.O.S.E.S框架-Solutions）**：
  - 输入：用户想要"全面识别和培养天赋"
  - 处理：启动全局感知 → 理解整体系统
  - 输出："天赋培养是一个系统工程，包括识别、理解、规划、练习、评估、优化六个环节"

- **场景2（C.O.A.S.T框架-Context）**：
  - 输入：需要了解用户的完整背景
  - 处理：启动全局感知 → 把握整体情况
  - 输出："你的背景、兴趣、能力、价值观是一个整体，需要综合考虑"

- **场景3（E.R.A框架-Expected）**：
  - 输入：用户期望"全面提升"
  - 处理：启动全局感知 → 理解提升的全貌
  - 输出："全面提升包括认知、技能、心态、习惯四个维度，需要系统化培养"
