# Function Calling 技术讲解与实战指南

## 任务目标
- 本 Skill 用于：系统讲解 Function Calling 技术的原理、架构设计和工程实践
- 能力包含：核心概念讲解、技术原理剖析、实战指导、最佳实践分享、以及使用技巧
- 触发条件：当用户需要调试或调用工具时、用户询问 Function Calling 概念、需要设计函数 Schema、遇到调用问题或寻求最佳实践建议

## 操作步骤

### 步骤 1：基础概念讲解
当用户需要了解 Function Calling 的基本概念时，智能体将：
1. 定义 Function Calling：解释它是 LLM 调用外部函数的机制
2. 阐述核心价值：说明如何解决知识陈旧和无法交互外部系统的问题
3. 说明工作流程：描述"用户请求→模型决策→工具执行→结果整合→最终响应"的完整循环
4. 对比传统方案：说明相比提示词工程的优势（标准化、准确率高、可维护性强）

关键要点：
- 模型不直接执行函数，只生成调用指令（函数名+参数）
- 客户端负责实际执行和结果返回
- 模型根据执行结果生成最终自然语言响应

### 步骤 2：技术原理讲解
当用户需要深入理解技术架构时，智能体将：
1. 讲解执行循环：消息上下文管理、模型决策、工具执行、结果注入的完整流程
2. 说明 JSON Schema 的作用：
   - 定义函数参数结构（类型、约束、描述）
   - 确保模型生成符合规范的参数
   - 实现函数定义和调用的标准化
3. 演示典型 Schema 示例：展示函数定义的 JSON 格式
4. 解释模型决策逻辑：模型如何判断是否需要调用函数、选择哪个函数、生成什么参数

核心概念：
- 工具列表：以 JSON Schema 格式向模型声明可用函数
- 消息上下文：包含用户历史对话、函数调用记录、工具执行结果
- 参数生成：模型根据 Schema 生成类型正确、语义合理的参数

### 步骤 3：实战指导
当用户需要实际应用 Function Calling 时，智能体将：

**函数设计指导**：
1. 功能单一化：每个函数只做一件事，职责清晰
2. 参数精简化：只包含必要参数，避免冗余
3. 命名语义化：使用清晰的动词-名词组合（如 get_weather, send_email）
4. 描述详尽化：提供清晰的功能说明、参数含义、返回值描述

**Schema 定义指导**：
1. 参数类型：根据实际需求选择 string、number、boolean、object、array
2. 必填字段：通过 required 数组明确标识必需参数
3. 枚举约束：使用 enum 限制参数取值范围
4. 嵌套结构：使用 properties 实现复杂对象定义

**错误处理指导**：
1. 参数验证：客户端验证模型生成的参数是否符合 Schema
2. 执行错误：捕获工具执行异常，返回明确的错误信息
3. 超时处理：设置合理的超时时间，避免长时间阻塞
4. 降级策略：当工具不可用时提供备选方案

### 步骤 4：进阶应用
当用户需要处理复杂场景时，智能体将：

**多轮对话处理**：
1. 上下文传递：将函数调用历史和结果持续维护在消息上下文中
2. 多次调用：支持单次请求调用多个函数
3. 并发执行：在无依赖关系时并发执行多个函数调用
4. 迭代调用：根据前一次函数结果决定是否需要调用其他函数

**复杂场景示例**：
- 用户查询天气并推荐穿搭：先调用 get_weather，再调用 recommend_clothing
- 预订行程：依次调用 check_availability、book_ticket、send_confirmation
- 数据分析：调用 query_database、calculate_statistics、generate_chart

### 步骤 5：最佳实践
当用户需要优化实现或避免常见问题时，智能体将提供以下指导：

**设计原则**：
1. 函数粒度：保持函数功能单一、参数精简
2. 命名规范：使用一致的命名约定（动词_名词形式）
3. 描述质量：提供清晰、准确、详尽的函数和参数描述
4. 安全防护：避免暴露敏感操作，实施权限控制

**性能优化**：
1. 并发调用：利用模型支持的多函数并发调用能力
2. 结果缓存：对频繁调用且结果稳定的函数实施缓存
3. 超时控制：设置合理的函数执行超时时间
4. 错误重试：对临时性错误实现指数退避重试机制

**安全考虑**：
1. 参数校验：严格验证模型生成的参数，防止注入攻击
2. 权限控制：根据用户身份限制函数调用权限
3. 审计日志：记录所有函数调用，便于追踪和调试
4. 敏感操作：对涉及数据修改的函数实施额外确认
