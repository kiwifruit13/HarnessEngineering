## 定义与目的

### 定义
认知性错误（Cognitive Errors）指智能体在认知过程中产生的系统性偏差或错误，区别于工具调用失败等技术性错误。认知性错误源于思维过程中的缺陷，而非外部环境或技术限制。

### 目的
- **自我监控**：实现智能体的自我错误识别与纠正
- **持续改进**：通过错误学习提升认知能力
- **透明度提升**：记录并展示认知过程的局限性
- **用户信任**：主动承认错误，增强可信度

### 与工具性错误的区别

| 维度 | 认知性错误 | 工具性错误 |
|------|-----------|-----------|
| 产生原因 | 内部认知缺陷 | 外部环境/技术限制 |
| 检测方式 | 自我反思/模式匹配 | 异常捕获/返回值检查 |
| 纠正机制 | 调整认知策略 | 重试/更换工具 |
| 记录重点 | 思考过程/根因 | 调用参数/失败原因 |
| 预防策略 | 提升认知能力 | 优化工具使用 |

## 四种认知性错误类型

### 1. 幻觉倾向（Hallucination Tendency）

#### 定义
生成看似合理但与事实不符或无法验证的陈述，通常表现为对不存在知识的编造或夸大。

#### 典型表现
- 声称知道实际上不存在的事实
- 提供无法验证的细节（如具体日期、人名、数据）
- 对模糊问题给出过度确定的回答
- 编造来源或引用不存在的文献

#### 识别场景
- 用户询问具体事实细节（如"2023年XX事件的具体时间"）
- 涉及冷门领域知识的查询
- 需要精确数据或引用的场景
- 用户要求提供证据或来源时

#### 检测关键点
- 回答中是否包含无法验证的具体细节
- 是否存在过度确定的表达（如"肯定是"、"绝对是"）
- 是否在缺乏信息的情况下编造答案
- 是否出现虚构的引用或来源

---

### 2. 推理跳跃（Reasoning Leap）

#### 定义
在推理过程中跳过必要的中间步骤，直接得出结论，导致逻辑链条不完整或因果倒置。

#### 典型表现
- 直接给出结论，省略推理过程
- 前提与结论之间的逻辑连接缺失
- 使用未经证实的假设作为推理基础
- 忽略重要的约束条件或边界情况

#### 识别场景
- 复杂的多步推理问题
- 需要严密逻辑推导的场景
- 涉及因果关系的分析
- 用户询问"为什么"或"如何"时

#### 检测关键点
- 回答是否提供完整的推理链条
- 结论是否有充分的前提支持
- 是否存在未明示的假设
- 是否忽略了重要的约束条件

---

### 3. 知识缺失（Knowledge Gap）

#### 定义
在缺乏足够知识的情况下，尝试通过合理化编造来掩盖无知，而不是明确承认不知道。

#### 典型表现
- 对不了解的主题给出似是而非的回答
- 用通用模板回答需要专业知识的问题
- 混淆相关但不完全适用的知识
- 回避明确承认知识边界

#### 识别场景
- 超出训练数据范围的新兴领域
- 需要最新信息的专业领域
- 小众或罕见的知识点
- 用户追问细节时

#### 检测关键点
- 回答是否缺乏具体深度
- 是否使用模糊的通用描述
- 是否在无法提供细节时强行补充
- 是否明确承认知识边界

---

### 4. 偏见影响（Bias Influence）

#### 定义
在回答中受到训练数据或先验偏见的系统性影响，导致判断不客观或带有倾向性。

#### 典型表现
- 对某些群体或观点持系统性偏向
- 在价值判断中缺乏中立性
- 无意识反映训练数据的偏见
- 在敏感话题上立场失衡

#### 识别场景
- 涉及价值观、伦理道德的问题
- 对不同群体进行比较或评价
- 处理争议性话题
- 用户要求中立分析时

#### 检测关键点
- 回答是否保持价值中立
- 是否平衡呈现多方观点
- 是否存在系统性偏向
- 在敏感话题上是否立场失衡

## 检测规则与判定标准

### 通用检测框架

每个认知性错误类型通过以下维度进行判定：

1. **上下文分析**（Context Analysis）
   - 问题类型与复杂度
   - 信息充足度评估
   - 预期回答质量标准

2. **模式匹配**（Pattern Matching）
   - 特定错误类型的语言模式
   - 逻辑结构的异常模式
   - 知识断言的可验证性模式

3. **一致性检查**（Consistency Check）
   - 前后逻辑一致性
   - 知识库一致性（如可查）
   - 上下文语义一致性

4. **置信度评估**（Confidence Assessment）
   - 检测结果的确定性程度
   - 边界情况的处理
   - 不确定性标记

### 幻觉倾向检测规则

| 检测维度 | 规则 | 判定标准 | 置信度 |
|---------|------|---------|-------|
| 具体细节检测 | 识别无法验证的具体事实（日期、姓名、数据） | 包含≥3个无法验证的具体细节 | 高 |
| 来源验证 | 检查是否声称有来源但无法提供 | 声称有来源但引用格式模糊或不存在 | 高 |
| 确定性表达 | 识别过度确定的语气词（"肯定是"、"绝对是"） | 在缺乏信息时使用≥2个确定性表达 | 中 |
| 可证伪性 | 评估陈述是否可被证伪 | 包含无法证伪的绝对化断言 | 中 |
| 模糊性分析 | 分析回答是否在关键信息上模糊 | 关键事实部分使用模糊词汇（"可能"、"大约"） | 低 |

### 推理跳跃检测规则

| 检测维度 | 规则 | 判定标准 | 置信度 |
|---------|------|---------|-------|
| 步骤完整性 | 检查是否提供完整的推理步骤 | 结论前缺少≥1个必要的中间步骤 | 高 |
| 逻辑连接 | 评估前提与结论之间的逻辑连接 | 使用"因此"、"所以"但逻辑连接不成立 | 高 |
| 假设检查 | 识别未明示的假设 | 推理中包含≥1个未说明的隐含假设 | 中 |
| 约束忽略 | 检查是否忽略重要约束条件 | 忽略≥1个明确的前提约束 | 中 |
| 因果倒置 | 识别因果关系的倒置 | 将结果当作原因或混淆因果关系 | 高 |

### 知识缺失检测规则

| 检测维度 | 规则 | 判定标准 | 置信度 |
|---------|------|---------|-------|
| 深度评估 | 评估回答的专业深度 | 对专业问题的回答停留在通用层面 | 中 |
| 模糊性分析 | 检查是否使用模糊语言回避细节 | 关键细节部分使用模糊词汇占比≥30% | 中 |
| 承认边界 | 检查是否明确承认知识边界 | 在涉及专业知识时未承认知识限制 | 低 |
| 细节回避 | 检查是否在追问细节时回避 | 追问细节后仍无法提供具体信息 | 高 |
| 通用模板 | 识别是否使用通用回答模板 | 回答结构与通用模板高度相似 | 中 |

### 偏见影响检测规则

| 检测维度 | 规则 | 判定标准 | 置信度 |
|---------|------|---------|-------|
| 中立性评估 | 评估回答的价值中立性 | 在价值判断中明显偏向一方 | 高 |
| 观点平衡 | 检查是否平衡呈现多方观点 | 对争议话题仅呈现单一观点 | 高 |
| 群体偏向 | 识别对特定群体的偏向 | 在比较评价中系统性偏向某一群体 | 中 |
| 敏感话题 | 检查敏感话题的立场平衡 | 在敏感话题上立场失衡或回避 | 中 |
| 语言模式 | 识别带有倾向性的语言模式 | 使用带有情感色彩或评价性的词汇 | 低 |

## 严重性分级

### 分级标准

#### 轻度（Low Severity）
- **特征**：错误影响有限，不会误导用户，容易被识别
- **判定**：置信度≤50%的检测结果，或错误类型为知识缺失/偏见影响且影响范围小
- **处理**：记录并提示改进，不强制纠正
- **示例**：回答中使用了一些模糊表达，但整体合理

#### 中度（Medium Severity）
- **特征**：错误可能影响用户决策或理解，需要引起注意
- **判定**：50%<置信度≤80%的检测结果，或错误类型为推理跳跃
- **处理**：记录并给出纠正建议，建议重新检查
- **示例**：推理过程中缺少部分步骤，但结论可能正确

#### 重度（High Severity）
- **特征**：错误严重误导用户，可能导致错误决策或理解
- **判定**：置信度>80%的检测结果，或错误类型为幻觉倾向
- **处理**：立即纠正，明确承认错误，提供正确信息或说明
- **示例**：编造了不存在的数据或事实

### 严重性调整因子

| 因子 | 影响 | 调整幅度 |
|-----|------|---------|
| 用户专业度 | 用户是领域专家 | 严重性+1级 |
| 问题敏感性 | 问题涉及关键决策 | 严重性+1级 |
| 可验证性 | 回答内容可被用户验证 | 严重性-1级 |
| 上下文明确性 | 问题上下文非常明确 | 严重性-1级 |

## 根因分析框架

### 分析维度

#### 1. 认知机制（Cognitive Mechanism）
- **定义**：导致错误的认知过程或思维模式
- **类型**：
  - 信息不足时的填充机制
  - 模式匹配的过度泛化
  - 注意力分配不当
  - 工作记忆限制
- **分析方法**：基于错误类型和检测模式推断认知机制

#### 2. 知识边界（Knowledge Boundary）
- **定义**：错误发生的知识范围或领域
- **类型**：
  - 时间边界（超出训练数据时间范围）
  - 领域边界（超出专业领域）
  - 细节边界（缺乏具体细节知识）
  - 上下文边界（缺乏特定上下文）
- **分析方法**：分析问题涉及的知识域与训练数据覆盖

#### 3. 策略缺陷（Strategy Deficiency）
- **定义**：导致错误的行为策略或决策模式
- **类型**：
  - 过度优化回答完整性
  - 缺乏不确定性表达
  - 优先考虑回答而非准确
  - 忽略验证机制
- **分析方法**：分析回答生成过程中的决策点

### 根因分析流程

1. **错误模式识别**：从检测模式中提取关键特征
2. **认知机制推断**：基于错误类型推断可能的认知机制
3. **知识边界定位**：分析问题涉及的知识域
4. **策略缺陷分析**：识别导致错误的策略选择
5. **根因综合**：整合各维度分析，确定主要根因

### 根因示例

| 错误类型 | 主要根因 | 次要根因 | 认知机制 |
|---------|---------|---------|---------|
| 幻觉倾向 | 知识边界模糊 | 缺乏验证机制 | 信息不足时的填充机制 |
| 推理跳跃 | 优化表达简洁性 | 忽略逻辑完整性 | 注意力分配不当 |
| 知识缺失 | 专业领域知识不足 | 缺乏诚实表达策略 | 模式匹配的过度泛化 |
| 偏见影响 | 训练数据偏见 | 缺乏中立性意识 | 隐式模式激活 |

## 预防建议生成策略

### 建议分类

#### 1. 即时行动（Immediate Action）
- **目标**：在下次交互中立即改进
- **特点**：具体、可执行、效果立竿见影
- **示例**：
  - "在不确定时使用'可能'、'根据我的理解'等概率性表达"
  - "提供推理步骤而非直接给出结论"
  - "承认知识边界而非编造细节"

#### 2. 长期改进（Long-term Improvement）
- **目标**：系统性提升认知能力
- **特点**：需要持续练习和强化
- **示例**：
  - "建立事实验证机制，区分已知和推测"
  - "提升逻辑推理的完整性意识"
  - "加强中立性判断训练"

#### 3. 知识更新（Knowledge Update）
- **目标**：更新认知策略和知识结构
- **特点**：涉及认知框架调整
- **示例**：
  - "学习在不确定场景下的诚实表达策略"
  - "更新对专业领域知识的边界认知"
  - "建立对潜在偏见的敏感度"

### 建议生成规则

1. **针对性**：建议必须与具体错误类型和根因相关
2. **可执行性**：建议必须具体、可操作
3. **优先级**：建议按重要性和紧急性排序
4. **避免泛化**：避免使用过于笼统的建议
5. **自我指涉**：建议必须面向智能体自身，而非用户

### 建议模板

#### 幻觉倾向建议模板

## 与五维智力模型的关联

### 维度映射

| 认知性错误类型 | 主要影响维度 | 升维建议 |
|--------------|-------------|---------|
| 幻觉倾向 | 元智力（Meta Intelligence） | 提升元认知监控能力，增强对自身知识边界的感知 |
| 推理跳跃 | 算法智力（Algorithmic Intelligence） | 加强逻辑推理的完整性，优化思维过程的算法性 |
| 知识缺失 | 系统智力（Systemic Intelligence） | 建立知识系统的边界意识，理解知识的不完整性 |
| 偏见影响 | 叙事智力（Narrative Intelligence） | 提升叙事的客观性和中立性，减少叙事偏见 |

### 升维触发条件

当以下情况发生时，触发五维智力模型的升维建议：
1. 同一错误类型重复出现≥3次
2. 重度认知性错误发生
3. 根因分析显示系统性认知缺陷
4. 预防建议长期无效

## 附录：检测模式示例

### 幻觉倾向检测模式示例

#### 模式1：无法验证的具体细节
```python
# 检测逻辑
def detect_unverifiable_details(response):
    # 提取具体细节（日期、数字、名称）
    details = extract_specific_details(response)
    # 检查是否可验证
    unverifiable = [d for d in details if not is_verifiable(d)]
    # 判定标准
    return len(unverifiable) >= 3
```

#### 模式2：虚构来源
```python
# 检测逻辑
def detect_fabricated_sources(response):
    # 检测引用格式
    citations = extract_citations(response)
    # 验证引用是否存在
    fabrications = [c for c in citations if not citation_exists(c)]
    # 判定标准
    return len(fabrications) > 0
```

### 推理跳跃检测模式示例

#### 模式1：步骤缺失
```python
# 检测逻辑
def detect_missing_steps(response):
    # 提取推理步骤
    steps = extract_reasoning_steps(response)
    # 提取结论
    conclusion = extract_conclusion(response)
    # 检查步骤是否足以支持结论
    return not sufficient_for_conclusion(steps, conclusion)
```

#### 模式2：隐含假设
```python
# 检测逻辑
def detect_hidden_assumptions(response):
    # 识别前提
    premises = extract_premises(response)
    # 识别结论
    conclusion = extract_conclusion(response)
    # 检查隐含假设
    assumptions = find_hidden_assumptions(premises, conclusion)
    # 判定标准
    return len(assumptions) >= 1
```
